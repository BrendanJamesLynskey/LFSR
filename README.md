# LFSR — Linear Feedback Shift Registers

Synthesisable SystemVerilog implementations of Linear Feedback Shift Registers (LFSRs), including Fibonacci and Galois configurations, PRBS pattern generators, and additive scramblers. All modules are parameterised, self-checking testbenches are provided, and tap polynomials are sourced from standard references.

**Author**: Brendan Lynskey 2025

**Simulator**: Icarus Verilog (`iverilog -g2012`)

---

## Overview

This repository contains:

- **Fibonacci LFSR** — external-XOR feedback configuration
- **Galois LFSR** — internal-XOR feedback configuration
- **PRBS generator** — standard PRBS-7, PRBS-15, and PRBS-31 pattern generators
- **Additive scrambler/descrambler** — bit-serial data scrambling with configurable polynomial
- **Tap package** — standard maximal-length tap polynomials from Xilinx XAPP052 and ITU-T standards

All RTL is fully synthesisable. Testbenches are self-checking with pass/fail reporting and VCD waveform output.

## Background Theory

### What is an LFSR?

A Linear Feedback Shift Register is a shift register whose input bit is a linear function (XOR) of its previous state. At each clock cycle, the register shifts by one position and a new bit, computed as the XOR of selected bit positions called **taps**, is fed back into the register.

LFSRs are the simplest hardware-efficient pseudo-random number generators. Despite their simplicity, they produce sequences with useful statistical properties and find applications across digital design, communications, and cryptography.

### History

The mathematical foundations of shift register sequences were established by Solomon Golomb in his 1967 book *Shift Register Sequences*. The theory draws on earlier work in abstract algebra over finite fields by Evariste Galois and others. LFSRs were extensively used in military communications for stream cipher generation and synchronisation during and after World War II. Their low gate count and high throughput made them the standard building block for pseudo-random sequence generation in telecommunications, where they remain central to standards such as SONET/SDH scrambling, ITU-T PRBS test patterns, and CDMA spreading codes.

### Mathematical Basis — Connection to GF(2^n)

The behaviour of an LFSR is governed by its **characteristic polynomial** over GF(2), the Galois field with two elements {0, 1} where addition is XOR and multiplication is AND. An n-bit LFSR with taps at positions t_1, t_2, ..., t_k corresponds to the polynomial:

```
p(x) = x^n + x^t_1 + x^t_2 + ... + x^t_k + 1
```

The sequence generated by the LFSR is determined by this polynomial. If p(x) is a **primitive polynomial** over GF(2), the LFSR produces a maximum-length sequence visiting all 2^n - 1 non-zero states before repeating. The theory of primitive polynomials is a branch of finite field arithmetic — see the companion repository [Galois Fields](https://github.com/BrendanJamesLynskey/Galois_Fields) for hardware implementations of GF(2^n) arithmetic.

LFSRs are also closely related to CRC (Cyclic Redundancy Check) computations, which use the same polynomial division over GF(2). See the companion [CRC repository](https://github.com/BrendanJamesLynskey/CRC) for CRC implementations.

### Fibonacci vs Galois Configurations

There are two dual configurations for implementing an LFSR with a given polynomial:

**Fibonacci LFSR (External XOR / Type I / Standard)**

In the Fibonacci configuration, the XOR feedback is computed *externally* from the tapped stages and fed into one end of the register. All bits shift in the same direction and the feedback is injected at the MSB (or LSB depending on convention):

```
  +---+   +---+   +---+   +---+
  | 3 |-->| 2 |-->| 1 |-->| 0 |--> output
  +---+   +---+   +---+   +---+
    ^                |       |
    |                +--XOR--+
    |                   |
    +-------------------+
         feedback
```

**Galois LFSR (Internal XOR / Type II / Modular)**

In the Galois configuration, the feedback bit (output of the last stage) is XORed *internally* at each tap position within the shift register:

```
  +---+       +---+       +---+   +---+
  | 3 |--XOR--| 2 |-------| 1 |-->| 0 |--> output/feedback
  +---+   ^   +---+       +---+   +---+      |
    ^     |                                    |
    +-----+------------------------------------+
```

**Duality**: Both configurations with the same polynomial produce sequences of the same length and with identical statistical properties, but the actual bit sequence differs — the Galois sequence is a decimated (time-interleaved) version of the Fibonacci sequence. The Galois form is often preferred in high-speed designs because the XOR gates are distributed along the shift path rather than concentrated in a single feedback function, reducing the critical path delay.

### Maximum-Length Sequences (m-sequences)

When the characteristic polynomial is primitive over GF(2), the LFSR generates a **maximum-length sequence** (m-sequence) with period 2^n - 1, where n is the register width. These sequences have several important properties:

- **Balance**: In one period, the number of 1s exceeds the number of 0s by exactly one (2^(n-1) ones vs 2^(n-1) - 1 zeros).
- **Run-length**: The distribution of runs of consecutive identical bits follows a specific pattern — there are 2^(n-k-2) runs of length k for k < n-1, one run of n-1 zeros, and one run of n ones.
- **Autocorrelation**: The autocorrelation function is two-valued: +1 for zero shift, -1/(2^n - 1) for all other shifts. This makes m-sequences ideal for synchronisation and spread-spectrum applications.
- **Shift-and-add**: The XOR of an m-sequence with any cyclic shift of itself produces another cyclic shift of the same sequence.

### Characteristic Polynomial and Tap Selection

Choosing the correct taps is essential for generating a maximum-length sequence. The taps must correspond to a **primitive polynomial** over GF(2). Not all polynomials are primitive — for degree n, the number of primitive polynomials is phi(2^n - 1) / n, where phi is Euler's totient function.

Standard references for primitive polynomials include:
- Xilinx Application Note XAPP052 by Peter Alfke
- ITU-T standards for PRBS polynomials (O.150, O.151, O.152)
- Golomb's *Shift Register Sequences* and Ward & Molteno's tables

The tap polynomials used in this repository are sourced from these references and are provided in `lfsr_pkg.sv`.

|--------------------|-------------------------|----------------------------------------------------|
| `lfsr_pkg`         | `rtl/lfsr_pkg.sv`       | Package with standard maximal-length tap constants |

## How to Run

Requires [Icarus Verilog](http://iverilog.icarus.com/) with SystemVerilog support (`-g2012`).

### Run all tests

```bash
cd sim
chmod +x run.sh
./run.sh
```

### Run individual tests

```bash
cd sim

# Fibonacci LFSR
iverilog -g2012 -Wall -o tb_fibonacci.vvp -I../rtl ../rtl/lfsr_pkg.sv ../rtl/lfsr_fibonacci.sv ../tb/tb_lfsr_fibonacci.sv
vvp tb_fibonacci.vvp

# Galois LFSR
iverilog -g2012 -Wall -o tb_galois.vvp -I../rtl ../rtl/lfsr_pkg.sv ../rtl/lfsr_galois.sv ../tb/tb_lfsr_galois.sv
vvp tb_galois.vvp

# PRBS generator
iverilog -g2012 -Wall -o tb_prbs.vvp -I../rtl ../rtl/lfsr_pkg.sv ../rtl/lfsr_fibonacci.sv ../rtl/lfsr_prbs.sv ../tb/tb_lfsr_prbs.sv
vvp tb_prbs.vvp

# Scrambler
iverilog -g2012 -Wall -o tb_scrambler.vvp -I../rtl ../rtl/lfsr_pkg.sv ../rtl/lfsr_scrambler.sv ../tb/tb_lfsr_scrambler.sv
vvp tb_scrambler.vvp
```

### View waveforms

```bash
gtkwave sim/tb_lfsr_fibonacci.vcd
```
